# v0.0.1-beta — Initial Release

## Overview

First public release of the Lux compiler — a compiled systems programming language targeting LLVM IR. Combines the performance of C with modern ergonomics, generics, and zero-cost C interop.

## What's New

### `fn` Keyword Syntax

Functions now use `fn` keyword syntax with return type after parameters:

```lux
fn add(int32 a, int32 b) int32 {
    ret a + b;
}

fn main() int32 {
    println(add(3, 4));
    ret 0;
}
```

`extend` methods follow the same pattern:

```lux
extend Point {
    fn distance(&self) float64 {
        ret sqrt(self.x * self.x + self.y * self.y);
    }
}
```

### Language

- **Type System**: `int8`–`int128`, `intinf`, `uint8`–`uint128`, `float32`–`float128`, `double`, `bool`, `char`, `string`, `cstring`, `void`, pointers, arrays, slices
- **Functions**: Named, generic with monomorphization, variadic, function pointers (`fn(T) -> U`)
- **Control Flow**: `if`/`else`, `for`-in, classic `for`, `while`, `do-while`, `switch`
- **Data Types**: `struct`, `enum` (unit/payload/named/generic), `union`, type aliases
- **Generics**: Parametric polymorphism with `<T>`, constraints, monomorphization
- **Extend Blocks**: Add methods to any type
- **Error Handling**: `try`/`catch`/`finally`, `throw`, `defer`, `?` propagate operator
- **Pointers**: Raw `*T`, `&`/`*`, `->`, arithmetic
- **Concurrency**: `spawn`/`await`, `Task<T>`, `Mutex`, `lock`
- **FFI**: `extern` declarations, `#include`, ABI-compatible structs, `cstring`

### Standard Library

20+ modules: `log`, `io`, `math`, `str`, `conv`, `fmt`, `fs`, `path`, `process`, `os`, `time`, `random`, `mem`, `hash`, `bits`, `ascii`, `regex`, `encoding`, `crypto`, `compress`, `net`, `thread`, `test`

### Collections

- `vec<T>` — dynamic array
- `map<K, V>` — hash map
- `set<T>` — hash set

### Tooling

- **LSP**: Semantic highlighting, hover info, code completion, signature help
- **CLI**: `build`, `run`, `check`, `test`, `helpc` commands
- **VS Code extension**: Syntax highlighting, LSP integration

### Compiler

- ANTLR4 parser frontend
- LLVM 22 IR backend
- Optimization levels `-O1` through `-O3`, `-Os`, `-Oz`
- LTO (Link Time Optimization)
- JIT execution mode
- Emit assembly, LLVM IR, or object files
